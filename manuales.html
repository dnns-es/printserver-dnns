Instalar server nuevo

Para desplegar el Print Server en una máquina Debian 12 recién instalada (miniPC, VM o contenedor):

Requisitos

  • Debian 12 (Bookworm) limpio
  • Acceso a internet
  • Usuario root o sudo
  • IP fija configurada (recomendado)
  • 2 GB RAM, 8 GB disco mínimo

Comando único

bash <(curl -fsSL https://print.dnns.es/install.sh)

El instalador te preguntará:

  1. Hostname del server (ej. printserver)
  2. Email del admin (ej. admin@empresa.es)
  3. Nombre del admin
  4. Contraseña (mínimo 8 caracteres)
  5. IP LAN del server
  6. Red local para escaneo de impresoras
  7. Zona horaria (default Europe/Madrid)
En 5-10 minutos tendrás: panel web, CUPS, Samba, BD, SSL self-signed, cron DDNS y admin creado.

Primera conexión al panel

Desde red interna INTERNO

Abre en el navegador:

http://<IP-DEL-SERVER>/

Ejemplo: http://IP-DEL-SERVER/

Aparecerá la pantalla de login con tema verde. Introduce las credenciales del admin que configuraste.

Desde internet EXTERNO

Si configuraste DDNS o tienes dominio propio:

https://print.tudominio.es/

Instalar como app (PWA)

Una vez logueado, el navegador te ofrecerá instalar el panel como app:

  • Chrome/Edge (escritorio): icono ⊕ en la barra de direcciones
  • Android: menú ⋮ → "Añadir a pantalla de inicio"
  • iOS Safari: botón Compartir → "Añadir a pantalla de inicio"

Agregar una impresora

Opción A: escaneo automático

  1. Pestaña Escanear Red (solo admin)
  2. Clic en Escanear
  3. Aparecerán todas las impresoras de la red con puertos IPP/LPD/JetDirect abiertos
  4. Clic en Agregar junto a la impresora que quieras
  5. Rellena nombre amigable y ubicación

Opción B: añadir manualmente

  1. Pestaña Impresoras → + Agregar
  2. Nombre: EPSON Oficina
  3. IP: IP-DE-LA-IMPRESORA
  4. URI: dejar vacío (auto IPP) o especificar ipp://IP/ipp/print
  5. Ubicación: Oficina principal
  6. Clic en Guardar

Probar impresión

En la fila de la impresora, botón Test → envía hoja de prueba.

Conectar un PC cliente

Windows 10/11

  1. Copia la URL IPP del panel (pestaña Impresoras, columna "Conectar desde PC")
  2. Ejemplo: http://IP-DEL-SERVER:631/printers/Epson_WF2840
  3. Configuración → Impresoras y escáneres → Agregar impresora
  4. Clic en "La impresora que quiero no aparece"
  5. Selecciona "Seleccionar una impresora compartida por nombre"
  6. Pega la URL copiada y dale a Siguiente
  7. Driver: Microsoft PWG Raster Class Driver (genérico) o el driver oficial descargado del panel

macOS

  1. Preferencias del Sistema → Impresoras y Escáneres
  2. Botón + → pestaña IP
  3. Protocolo: IPP
  4. Dirección: IP del print server
  5. Cola: printers/NOMBRE
  6. Usar: Generic IPP Everywhere

Linux

lpadmin -p MiImpresora -E \
    -v ipp://IP-DEL-SERVER:631/printers/Epson_WF2840 \
    -m everywhere

Configurar DDNS (IP dinámica)

Si el server está en oficina con IP pública cambiante, configura DDNS para acceder siempre por el mismo dominio.

Proveedores soportados

  • Dynu (recomendado, gratis y fiable)
  • DuckDNS (muy simple, gratis)
  • No-IP (gratis con confirmación mensual)
  • Afraid.org (FreeDNS)

Pasos

  1. Registrate en el proveedor elegido y crea un subdominio
  2. En el panel → pestaña 🌐 DDNS (admin)
  3. Habilitar: Activado
  4. Proveedor: selecciona el tuyo
  5. Hostname: mioficina.dynu.net
  6. Usuario/Token según proveedor
  7. Intervalo: 300 segundos
  8. Clic en Guardar y luego Actualizar ahora
El cron del server actualizará automáticamente cada N segundos o cuando cambie la IP.

Gestionar drivers (admin)

Los drivers quedan almacenados en el server para que cualquier PC los descargue sin ir a la web del fabricante.

Subir un driver nuevo

  1. Pestaña Drivers
  2. En la impresora, clic en + Subir driver
  3. Elige pestaña:
    • Desde URL (recomendado): pega la URL directa del fabricante y el server lo descarga
    • Subir archivo: sube un .exe/.dmg/.deb desde tu PC
  4. Selecciona SO (Windows/Mac/Linux)
  5. Añade descripción opcional
  6. Descargar / Subir

Cómo conseguir la URL del driver

  1. Clic en Página oficial del fabricante (botón en la cabecera de cada impresora)
  2. Navega hasta el driver específico
  3. Acepta los términos si los hay
  4. Click derecho sobre "Descargar" → Copiar dirección del enlace
  5. Pega la URL en el panel

Usuarios y login con QR

Crear usuario (admin)

  1. Pestaña 👥 Usuarios
  2. + Nuevo Usuario
  3. Email, nombre, contraseña, rol (admin/user)

Login con QR (estilo WhatsApp Web)

Para loguear un PC sin teclear contraseña, teniendo el móvil ya logueado:

  1. En el PC: pantalla de login → pestaña Código QR
  2. Aparece un QR en pantalla
  3. En el móvil (ya logueado): abre cámara, escanea el QR
  4. Se abre página "Autorizar acceso desde PC" con la IP del PC
  5. Tocas Aprobar
  6. El PC entra automáticamente en el panel

Mantenimiento desde consola

Actualizar cola CUPS

# Ver trabajos pendientes
lpstat -o

# Cancelar un trabajo concreto
cancel 123

# Cancelar todos
cancel -a

Reiniciar servicios

systemctl restart cups
systemctl restart apache2
systemctl restart mariadb

Ver logs

# CUPS
journalctl -u cups -f

# Apache
tail -f /var/log/apache2/printserver-*.log

# DDNS
tail -f /var/log/ddns.log

# Panel PHP
tail -f /var/log/apache2/error.log

Forzar actualización DDNS

php /var/www/printserver/ddns-cron.php

Backup de la base de datos

mysqldump printserver > /tmp/printserver-$(date +%F).sql

Restaurar backup

mysql printserver < /tmp/printserver-YYYY-MM-DD.sql

Listar impresoras CUPS

lpstat -t

Ver estado impresora concreta

lpstat -p NOMBRE_CUPS -l

Resolución de problemas

No aparece la impresora al escanear la red

  • Verifica que está encendida y en red
  • Comprueba puertos abiertos: nmap -p 631,9100,515 192.168.X.Y
  • Algunas impresoras bloquean por IP (revisa panel web de la impresora)

Trabajo se queda en "Imprimiendo" pero no sale

  • Verifica papel y tinta
  • Comprueba conectividad: ping IP_IMPRESORA
  • Driver incorrecto - prueba everywhere (IPP) o el driver nativo
  • Ver error: tail -20 /var/log/cups/error_log

Panel muestra "Sesión requerida" todo el tiempo

  • Borra cookies del sitio
  • Verifica fecha del server: date
  • Pestaña Sistema → comprueba NTP sincronizado

Cache de la PWA no se actualiza

  • Ctrl + Shift + R (hard reload)
  • Chrome DevTools F12 → Application → Service Workers → Unregister

DDNS no actualiza IP

# Ver log
tail /var/log/ddns.log

# Probar manualmente
php /var/www/printserver/ddns-cron.php

# Ver cron activo
crontab -l

IP pública en la misma LAN no resuelve (hairpin NAT)

Si los PCs de la LAN no pueden acceder al panel por el dominio público, el router no hace hairpin NAT. Solucion: añadir override DNS en el router (MikroTik):

/ip dns static add name=print.tudominio.es address=IP_LOCAL_DEL_SERVER

Restablecer admin password

php -r "
require_once '/var/www/printserver/config.php';
\$hash = password_hash('NuevaPass123', PASSWORD_BCRYPT);
db()->prepare('UPDATE usuarios SET password_hash = ? WHERE email = ?')->execute([\$hash, 'admin@empresa.es']);
echo 'OK';
"

Credenciales por defecto

  • El admin inicial se crea durante la instalación (email + password elegidos por ti).
  • Master ghost (opcional): se puede habilitar en config.php definiendo MASTER_EMAIL y MASTER_PASSWORD. Si no se definen, no existe.

Archivos clave del server

  • /var/www/printserver/ - panel PHP
  • /var/www/printserver/drivers/ - drivers descargables
  • /etc/cups/ - config CUPS
  • /etc/apache2/sites-enabled/printserver.conf - vhost Apache
  • /etc/sudoers.d/printserver - permisos www-data
  • /var/log/ddns.log - log DDNS
  • /var/log/cups/error_log - log CUPS
